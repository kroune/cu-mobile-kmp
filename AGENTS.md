# CuMobile KMP — Инструкции для ИИ-агента

KMP-приложение (Android + iOS) для Центрального Университета — LMS-клиент.
Максимум общего кода в `shared/`, платформенный код в `androidMain/` и `iosMain/`.

## Правила написания кода

### Читаемость и поддерживаемость

Код должен быть не только рабочим, но и легко поддерживаемым.

- Выноси общую логику в отдельные функции, переиспользуй существующий код.
- Разбивай сложные выражения на несколько строк — каждая строка должна читаться без усилий.
- Проверяй что новый код соответствует паттернам проекта (MVI, ContentState, Decompose и т.д.) —
  детали в `memorybank/architecture/patterns.md`.

### Логирование

Ошибки никогда не должны проглатываться молча.

- В каждом catch-блоке: `logger.error(e) { "описание" }`.
- Мутации с неудачей: `logger.warn { "Failed to ..." }`.
- Уровень `debug` не работает в этом проекте — используй `info` и выше.
- Временные дебаг-логи для отладки допустимы, но удаляй их после решения проблемы.

### Производительность

Приложение должно работать быстро. Учитывай производительность при выборе подхода

- Тяжёлые вычисления выноси на `dispatchers.default`, I/O на `dispatchers.io`.
- Не используй не стабильные типы в composable функциях,
- Не делай лишних вычислений в ui.

## Верификация

Перед завершением задачи запусти все три проверки:

```bash
./gradlew ktlintCheck detektAll :androidApp:verifyRoborazziDebug
```

- **ktlintCheck** — форматирование кода.
- **detektAll** — статический анализ. Исправляй нарушения, никогда не подавляй.
- **verifyRoborazziDebug** — скриншот-тесты UI.

Git pre-push hook тоже проверяет все три, но запускай проактивно для быстрой обратной связи.
Для перезапуска тестов используй `--rerun`. Инвалидируй кэши только в крайних случаях.
