Цель проекта - написание kmp приложения для лмс ЦУ (центральный университет). Поддержка android, ios. Предпочтительно шарить большую часть кода между двумя платформами.

Сейчас уже есть рабочая версия на flutter (https://github.com/cu-3rd-party/lms-mobile), он уже проиндексирован deepwiki, а так же установлен в /home/olowo/StudioProjects/lms-mobile. В случае сложностей с запуском, недоступностью mcp - обращайся ко мне. Ты должен аккуратно сделать fork, не сломав функционал. Этот компьютер на linux, в силу этого ты будешь тестировать на android, но важно закладываться на 2 платформы и хранить по минимуму логики в android/ios части.

Важно писать код на современном kotlin. Вот список предпочитаемых библиотек для каждого из возможных сценариев:
serialization/deserialization - kotlinx seralization
http client - ktor
логирование - logback
асинхронность - kotlinx coroutines
навигация для android - decompose
ui для ios/android - compose

Важно чтобы код в проекте не просто работал, но и хорошо поддерживался Вот некоторые из правил которым точно стоит следовать:
Важно переиспользовать функции и выносить общую логику в отдельные функции или использовать oop.
Файлы не должны быть больше 400 строк.
При создание/редактирование файлов надо проверять что это изменение соответствует архитектуре проекта.
В одной строчке кода не должно быть сложной логики т.к. это ухудшает читаемость.
Убедись что у всех функций/переменных стоит правильный модификатор доступ. При написание кода будь очень внимательным и аккуратным.
Инвалидируй кэши только в крайних случаях, в 99% случаев их наличие никак не мешает. Для перезапуска тестов используй опцию --rerun (а не --rerun-tasks).
Если ты пишешь план - обязательно сохраняй его как md файл.
Важную информацию сохраняй в memorybank/

Если ты неуверен в синтаксисе библиотеки - используй mcp deepwiki.
Для взаимодействия с проектом предпочитай mcp/tools calls, а не команды в терминале.

В конце своей работы перепроверь что весь код написанный тобой соответствует всем требованиям и отрефактори его при необходимости.
